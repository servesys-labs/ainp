## Step 3: Fill Out Submission Form

### Basic Information

**Title**:

```
AI-Native Network Protocol (AINP) for Semantic Agent Communication
```

**Abstract** (copy from draft):

```
This document specifies the AI-Native Network Protocol (AINP) version 0.1, a semantic communication protocol designed for intent exchange between AI agents. AINP replaces location-based routing with semantic routing, byte-stream delivery with intent delivery, and simple handshakes with multi-round negotiation. AINP enables agents to discover each other by capability rather than network location, negotiate terms autonomously, and exchange structured intents with cryptographic security.
```

## Step 5: After Submission

### Immediate

- You'll receive a confirmation email
- Draft will be assigned an official name (e.g., `draft-prasad-ainp-protocol-00`)
- Draft will be available at: `https://datatracker.ietf.org/doc/draft-prasad-ainp-protocol-00/`

### Within 24 Hours

- Draft will appear in the IETF Internet-Drafts repository
- Available at: `https://www.ietf.org/archive/id/draft-prasad-ainp-protocol-00.txt`
- HTML version: `https://www.ietf.org/archive/id/draft-prasad-ainp-protocol-00.html`

### Review Process

**Independent Submission Stream**:

- Independent Submissions Editor (ISE) will review
- May request revisions
- Timeline: 2-4 weeks for initial review

**Community Feedback**:

- Anyone can comment on the draft
- You can respond to comments
- Update draft with revisions (`-01`, `-02`, etc.)

## Step 6: Update Draft (Future Versions)

When you need to update:

1. **Edit**: `draft-ainp-protocol-00.md`
2. **Regenerate XML**:
   ```bash
   kramdown-rfc2629 docs/rfcs/draft-ainp-protocol-00.md > docs/rfcs/draft-ainp-protocol-01.xml
   ```
3. **Update version**: Change `-00` to `-01` in filename and docname
4. **Submit**: Upload new XML file via Datatracker

## Important Notes

### Draft Expiration

- Drafts expire **6 months** after submission
- Must update (`-01`, `-02`, etc.) before expiration
- If expired, must resubmit as new draft

### Draft Name Format

IETF assigns names like:

- `draft-{lastname}-{topic}-{version}`
- Example: `draft-prasad-ainp-protocol-00`

You cannot choose the exact name - it's based on your account.

### Submission Streams

**Independent Submission** (Recommended):

- ✅ Faster review
- ✅ Less bureaucracy
- ✅ Good for solo developers
- ⚠️ Less community review

**IETF Working Group**:

- Requires WG adoption
- More community review
- Slower process
- Better for standards track

## Troubleshooting

### "Invalid XML" Error

- Check XML file opens correctly
- Verify kramdown-rfc conversion worked
- Try regenerating XML: `kramdown-rfc2629 draft-ainp-protocol-00.md > draft-ainp-protocol-00.xml`

### "Draft Name Already Exists"

- IETF will auto-assign a name
- You can't control the exact name
- Just proceed with submission

### "Author Email Mismatch"

- Use the email associated with your Datatracker account
- Or add the email as an author in the draft
